# 🐻 FortuneBear (포춘베어)
> **"오늘 이건 조심해!" — AI 기반 위트 있는 조언 및 양자택일 의사결정 서비스**

FortuneBear는 기존의 막연한 운세 서비스에서 벗어나, 오늘 하루 주의해야 할 점을 자연스럽게 예고해주고 고민되는 두 가지 선택지 중 하나를 AI가 명쾌하게 골라주는 웹 서비스입니다. 

---

## ✨ 핵심 기능 (Core Functions)

### 1. Daily Caution (오늘의 예고)
- "오늘은 이런 점을 조심하면 좋아요"라는 컨셉으로 리스크를 유쾌하게 관리할 수 있는 조언을 제공합니다.
- 단순한 운세를 넘어 사용자에게 실질적이고 위트 있는 피드백을 전달합니다.

### 2. AI Decision Maker (양자택일)
- 짜장면 vs 짬뽕처럼 결정하기 힘든 두 가지 선택지 중 하나를 AI가 골라줍니다.
- 실시간 스코어 프로세싱을 통해 선택의 확신도를 시각화하여 재미를 더했습니다.

---

## 🛠 첫 AI 연동과 기술적 도전 (First AI Integration)

이 프로젝트는 저의 **첫 번째 AI API 연동 프로젝트**로, 모델의 특성을 파악하고 데이터의 신뢰성을 높이는 데 집중했습니다.

### 🔍 트러블슈팅: AI의 선택지 편향 문제 (Selection Bias)
AI 모델이 두 가지 선택지를 입력받았을 때, 논리적 근거와 관계없이 구조적으로 **첫 번째 옵션을 더 자주 선택하는 편향 현상**을 발견하고 이를 해결했습니다.

* **문제 분석:** 프롬프트 구조상 먼저 제시된 정보에 가중치를 더 두는 LLM의 특성 확인.
* **해결 방법 (Randomized Logic):**
    - 사용자 데이터를 API로 전송하기 전, 내부 로직에서 선택지의 순서를 무작위로 섞는 **셔플링 알고리즘** 도입.
    - **Temperature 제어:** 모델의 확산적 사고를 돕기 위해 온도를 미세 조정하여 데이터 근거 중심의 판단 유도.
* **결과:** 동일 질문에 대해 순서를 바꿔가며 테스트를 수행하여 선택 확률이 $50:50$에 수렴하도록 보정 완료.

## 🚀 주요 성과 및 경험
- **AI 연동 역량:** LLM API를 실제 서비스 비즈니스 로직에 성공적으로 결합.
- **데이터 정교화:** AI가 산출하는 점수 분포를 분석하여 사용자 경험에 최적화된 스코어 범위 구축.
- **캐릭터 브랜딩:** '운세 곰돌이' 마스코트를 활용하여 AI의 조언을 친근하게 전달하는 UI/UX 설계.
